## The Problem with the Standard Framing

The Hodge conjecture is stated in algebraic geometry: "every Hodge class
is algebraic." This SOUNDS like you need to construct a specific subvariety,
which is an existential/geometric problem. Hard to brute force.

**But**: we can REFRAME it into domains where systematic exhaustion IS possible.

## Reframe 1: Hodge as a LINEAR ALGEBRA Problem

### Setup
Fix a projective variety X of dimension n. The Hodge decomposition gives:

  H^{2p}(X, Q) = vector space of dimension b_{2p} (Betti number)
  H^{p,p}(X) = subspace of dimension h^{p,p}

The Hodge classes: Hdg^p(X) = H^{2p}(X, Q) ∩ H^{p,p}(X)

The algebraic classes: Alg^p(X) = Q-span of cycle classes of codim-p subvarieties

**Hodge conjecture**: Alg^p(X) = Hdg^p(X) as Q-vector spaces.

### The Brute Force
Both spaces are FINITE-DIMENSIONAL Q-vector spaces.

dim_Q(Hdg^p(X)) = ρ_p (the Hodge number of algebraic type)
dim_Q(Alg^p(X)) = rank of Néron-Severi group (generalized)

**Hodge ⟺ ρ_p = dim(Alg^p)**: the algebraic classes span ALL Hodge classes.

For SPECIFIC varieties: this is a FINITE computation.
1. Compute H^{2p}(X, Q) (topology: singular cohomology)
2. Compute the Hodge filtration (analysis: period integrals)
3. Identify Hdg^p(X) (linear algebra: intersect Q-structure with (p,p) part)
4. Enumerate known algebraic cycles (geometry: subvarieties)
5. Check if their classes span Hdg^p(X) (linear algebra: rank computation)

**If they do**: Hodge verified for this X at this p. Certificate.
**If they don't**: either we missed some cycles, or Hodge is FALSE for this X.

### The Generator
For each variety X in a systematically enumerable family:
  - Compute the period matrix Ω (numerical, to high precision)
  - Identify the Hodge lattice Hdg^p (exact, from Ω)
  - Enumerate all "obvious" algebraic cycles (products, intersections, pullbacks)
  - Check if they span Hdg^p

This is a BRUTE FORCE SEARCH over:
(a) The space of varieties (enumerable: degree d, ambient P^n, ...)
(b) The space of cycles on each variety (enumerable: subvarieties of each degree)

## Reframe 2: Hodge as a GROUP THEORY Problem

### The Mumford-Tate Group
For a Hodge structure V, the Mumford-Tate group MT(V) ⊂ GL(V_Q) is the
smallest Q-algebraic group whose real points contain the image of the
Hodge cocharacter h: U(1) → GL(V_R).

**Key theorem (Deligne)**: The Hodge classes in V^⊗ are exactly the
MT(V)-invariants: Hdg(V^⊗) = (V^⊗)^{MT(V)}.

**The Hodge conjecture for abelian varieties**: is equivalent to saying
the Mumford-Tate group determines the algebraic classes. Specifically:

  Hodge ⟺ every MT(V)-invariant tensor is algebraic

### The Brute Force in Group Theory
The Mumford-Tate group is a REDUCTIVE algebraic group over Q. These are
CLASSIFIED (Cartan classification: types A, B, C, D + exceptional).

For each possible MT group G ⊂ GL(V):
1. Compute G-invariants in V^⊗k for each k (representation theory)
2. These are the predicted Hodge classes
3. Check which are algebraic (construct the cycles)

**The enumeration**: For abelian varieties of dimension g:
- V = H^1(A, Q) has dimension 2g
- MT(A) ⊂ Sp(2g, Q) (symplectic group, from the polarization)
- The possible MT groups are classified by their type:
  - Generic: MT = Sp(2g) → only obvious Hodge classes
  - CM type: MT is a torus → many Hodge classes, all known to be algebraic for g ≤ 3
  - With real multiplication: MT is intermediate
  - With quaternionic multiplication: MT is smaller

**For each type of MT group**: check if all invariants are algebraic.
This is a FINITE CHECK for each group type.

### The Breakthrough for Abelian Varieties

The Hodge conjecture for abelian varieties reduces to:

**For each reductive Q-group G ⊂ Sp(2g), are all G-invariant tensors
in H^*(A, Q)^⊗ generated by algebraic cycles?**

This is:
- FINITE for each g (finitely many group types)
- CHECKABLE for each group type (representation theory)
- PROVED for g ≤ 3 (case-by-case through the classification)

**The generator**: enumerate MT groups by dimension, compute their
invariants, check algebraicity. Systematic and exhaustive.

## Reframe 3: Hodge as a LATTICE Problem

### The Hodge Lattice
H^{2p}(X, Z) is a lattice (free Z-module with an intersection form).
The Hodge structure gives a decomposition of H^{2p} ⊗ C.

A Hodge class is a lattice point in H^{2p}(X, Z) that lies in the
(p,p)-part of the Hodge decomposition (after tensoring with C).

**Algebraic classes form a SUBLATTICE** of the Hodge lattice.

**Hodge conjecture**: the sublattice of algebraic classes has the same
RANK as the Hodge lattice (after tensoring with Q).

### The Brute Force in Lattice Theory
For specific varieties: the Hodge lattice is COMPUTABLE (from periods).
The algebraic sublattice is ENUMERABLE (from known cycles).

**Algorithm**:
1. Compute the Hodge lattice L_Hdg (rank = ρ)
2. Enumerate algebraic cycles, compute their classes, get sublattice L_Alg
3. Compute rank(L_Alg)
4. If rank(L_Alg) = ρ → Hodge holds for this variety ✓
5. If rank(L_Alg) < ρ → search for more cycles (or: potential counterexample)

For K3 surfaces: ρ ≤ 20 (Picard number), and the Hodge lattice in H^{1,1}
is exactly the Néron-Severi lattice (by Lefschetz 1,1). This is PROVED.

For higher-dimensional varieties: the lattice computation is still possible
but the cycle enumeration becomes the bottleneck.

## Reframe 4: Hodge as a PERIOD DOMAIN Problem

### The Period Map
For a family of varieties {X_t}_{t∈S}: the Hodge structure on H^*(X_t)
varies with t. The period map Φ: S → D maps the parameter space to the
period domain (classifying space of Hodge structures).

A Hodge class on X_t is a section of a local system that stays of type (p,p)
at the point t ∈ S.

**Cattani-Deligne-Kaplan theorem (1995)**: The locus where a flat section
becomes a Hodge class is an algebraic subvariety of S.

This means: Hodge classes don't appear "randomly" — they appear in
algebraically parametrized families. This is STRUCTURAL EVIDENCE for Hodge.

### The Brute Force in Analysis
For a family {X_t}:
1. Compute the period map Φ(t) (system of ODEs: Picard-Fuchs equations)
2. Identify the Hodge locus (where flat sections become type (p,p))
3. Verify the Hodge locus is algebraic (CDK theorem: it always is)
4. On the Hodge locus: construct the algebraic cycle

Step 4 is the conjecture. But steps 1-3 are DOABLE by analysis (ODE solving)
and provide the right framework.

## The Master Generator

Combining all reframes:

```python
def hodge_brute_force():
    for family in enumerate_algebraic_families():
        for variety X in family:
            # Step 1: Compute periods (analysis)
            Omega = compute_period_matrix(X)
            
            # Step 2: Identify Hodge classes (linear algebra)
            Hdg = hodge_classes_from_periods(Omega)
            
            # Step 3: Compute Mumford-Tate group (group theory)
            MT = mumford_tate_group(Omega)
            
            # Step 4: Enumerate algebraic cycles (geometry)
            cycles = enumerate_cycles(X, max_degree=D)
            
            # Step 5: Check spanning (lattice theory)
            Alg = span(cycle_classes(cycles))
            
            if rank(Alg) == rank(Hdg):
                yield Certificate(X, "Hodge verified")
            else:
                yield GapReport(X, rank(Hdg) - rank(Alg), "missing cycles")
```

Each component uses a DIFFERENT mathematical domain:
- Step 1: Complex analysis (period integrals, Picard-Fuchs ODE)
- Step 2: Linear algebra over Q (lattice computation)
- Step 3: Algebraic group theory (Mumford-Tate classification)
- Step 4: Algebraic geometry (cycle enumeration)
- Step 5: Lattice theory (rank comparison)

The generator is EXHAUSTIVE: it will eventually check every variety in
every family. If Hodge is true: every run produces a certificate. If
Hodge is false: the generator finds the counterexample (a variety with
rank(Alg) < rank(Hdg) that persists no matter how many cycles we add).

existence problem.
