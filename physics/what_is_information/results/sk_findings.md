## Setup

The S/K bifurcation (attempt_001) predicts: Shannon entropy H (channel capacity, S-information)
and Kolmogorov complexity K (compressibility / structure, K-information) are **orthogonal** —
neither implies the other. The anti-problem (is random noise "maximal information" or "no information"?)
has a clean answer under the bifurcation: noise is S-maximal AND K-maximal. Random is the
worst case for both.

gzip / bzip2 / lzma are used as computable K-proxies (lower bounds on K via local-pattern
compression). Their ratios (compressed_size / original_size) approximate K: 0 = perfectly
compressible (K-poor), 1 = incompressible (K-rich).

## Finding 1: Quadrant structure confirmed

All expected quadrants are populated:

- **Hi-S/Hi-K:** random_bytes, base64_random — no structure, maximum uncertainty
- **Hi-S/Lo-K:** english_prose, source_code, rle_random_runs, incremental_uint32 — high uncertainty *and* compressible regularity
- **Lo-S/Lo-K:** periodic strings, morphic sequences — predictable and compressible
- **Lo-S/Hi-K:** EMPTY, as predicted — if a string is predictable (Lo-S), its distribution is concentrated, which means it has a short description (Lo-K). High K requires high variability.

The empty quadrant is structurally necessary, not accidental.

## Finding 2: The math constants — sharpest illustration of gzip-K ≠ true-K

π, e, √2 all produce:
- H ≈ 3.321 bits/byte — the theoretical maximum for the decimal digit alphabet (log₂(10))
- gzip ≈ 0.50, bzip2 ≈ 0.44, lzma ≈ 0.45 — moderate compression (not random, not structured)
- True K = O(1) — the program "compute π to N digits" is under 1KB for any N

The compressors reach gzip ≈ 0.50 because they detect the restricted alphabet ({0..9} maps to byte range 48–57) and achieve roughly 1.6× compression from that alone. But they cannot find the global structure — the fact that these 10 000 digits are generated by a finite algorithm. True K is O(log N) at worst (N appears as a parameter) and O(1) if N is hardcoded.

**This is the gzip-K blind spot:** any string generated by a short, deterministic algorithm with locally-random-looking output (π, LCG, cryptographic hashes) will have high gzip-K but low true-K. The compressors are measuring local recurrence depth; true K measures global description length.

**Consequence for physics:** the "information content" of a physical law like the Schrödinger equation or the Einstein field equations is K-information in this sense — a short global description that generates arbitrarily complex local behavior. The S-content of the observable states is high; the K-content of the laws is very low. This is the strongest case for K-monism in physics.

## Finding 3: The LCG adversarial — K-proxy failure in the other direction

The LZ77-adversarial generator uses a 32-bit linear congruential generator (state = 1664525·state + 1013904223 mod 2³²). Output:
- H = 7.980 bits/byte — virtually identical to true random (7.982)
- gzip = 1.002 — incompressible to gzip (larger than input due to headers)
- True K ≈ 100 bytes (the LCG spec itself)

gzip overestimates K for the math constants (fails to find global pattern); it also overestimates K for the LCG adversarial. Both are K=O(1) strings masquerading as K=MAX strings to any local-window compressor. The true-K difference between random bytes and LCG output is enormous (K=N bits vs K=100 bytes), but gzip cannot see it.

**Consequence for algorithmic information theory:** the Church-Turing / physical Church-Turing claim (attempt_001 in what_is_computation) that every physical regularity has a finite K-description is consistent with all these generators — they all have short specs. The gzip-K measure is a poor witness to this claim; it can tell you that *locally random* strings are *locally* K-rich, but it cannot attest to global K-poverty. Physical Church-Turing is a claim about global K, not local K.

## Finding 4: DNA random — K-proxy failure via alphabet artifact

Random DNA (uniform over A/T/C/G):
- H = 2.000 bits/byte — exact maximum for 4-symbol alphabet
- gzip = 0.321 — compresses to 32% of original
- True K = HIGH — it's genuinely random content, no structure

gzip compresses it to ~0.32× because gzip detects the restricted alphabet: only 4 distinct byte values appear. From gzip's perspective, this looks structured. But the content is uniformly random over those 4 values — the "compression" is entirely alphabet-level, not content-level.

This is the inverse of the math-constant case: gzip underestimates K for restricted-alphabet random strings (finds fake structure), while it overestimates K for short-program globally-structured strings (misses real structure).

**Both failures have the same root cause:** gzip is a local-recurrence compressor. It finds structure only when the same byte sequences recur within a sliding window (~32KB). Neither global algorithmic regularity nor restricted-alphabet randomness is captured faithfully.

## Finding 5: RLE random runs — S-high, K-low

Random-length runs of random bytes:
- H = 7.596 bits/byte — high, but not maximum (runs reduce global unpredictability slightly)
- gzip = 0.153, bzip2 = 0.203, lzma = 0.123 — excellent compression
- True K = medium (the run-generation rule is simple but the specific runs are random)

This lands in Hi-S/Lo-K: the byte-level distribution is near-random, but the local structure (repeated bytes within each run) is highly compressible. This is the case the theory predicts should exist: a string that is maximally uncertain at the macro level but locally structured. RLE-compressed audio waveforms, textures with coherence, biological sequences with tandem repeats all look like this.

## Implications for the three residual questions

**R1 — What physical quantities bound K in a region?**
The holographic bound bounds S (number of distinguishable states per unit area). These numbers confirm that gzip-K and S are not correlated in any simple way: rle_random_runs has H=7.6, gzip=0.15 (high S, low gzip-K); pi_digits has H=3.3, gzip=0.51 (medium S, medium gzip-K, low true-K). There is no S→K implication. A physical bound on K would require a different argument — not one inherited from the holographic principle. This supports the claim in gap.md that R1 is genuinely open.

**R2 — Wheeler's "it from bit": S-ontology or K-ontology?**
If physical laws are like π — short programs generating high-S observable outputs — then "reality is information" in the K sense, not the S sense. The holographic S-count of a region tells you how many distinguishable states it can have; the K-content of the laws tells you how much regularity governs which of those states actually occur. Wheeler's claim is most defensible as a K-claim: the regularity of physical laws (very low K) is what's fundamental, not the state count (high S). The math constants give numerical ground for this: you would never reconstruct π from an S-count of its digits; you need the generating program.

**R3 — Thermodynamic cost (S-erasure) vs cognitive value (K-accumulation)**
A first numerical handle: english_prose has H=4.13 bits/byte but gzip=0.044 (97% compressible). This means ~4 bits of S-content per byte, but only ~0.18 bits of gzip-K per byte. If we take gzip ratio as a rough K-content density, then reading English prose costs ~4 S-bits per byte processed (Landauer cost at each token boundary) but only accumulates ~0.18 K-bits per byte (the regularity that compressors capture). The ratio is ~22×: S-cost is 22× the K-gain, just from the compression accounting.

For source code: H=4.50, gzip=0.041 → ratio ≈ 22× as well. For random bytes: H=7.98, gzip=1.00 → ratio = 1 (S-cost = K-cost, nothing to learn). For periodic strings: H=1.0, gzip=0.005 → ratio ≈ 200× (very low S-cost, almost zero K-gain because K is already near zero — nothing left to compress).

The 20-25× S/K ratio in natural-language-like content is a concrete approximation to R3's claim. It is not a thermodynamic efficiency — it's a compression accounting that equates S-cost with "distinguishable state processing" and K-gain with "regularity extracted." Turning this into actual thermodynamic numbers requires connecting gzip-K bits to real K-information bits, which requires a better K-approximation than gzip (see next steps).

## Next numerical steps

1. **Better K-proxy for math constants.** Compute string complexity via the Lempel-Ziv 78 algorithm (LZ78 complexity = number of distinct phrases), which gives a different approximation to K. LZ78 complexity should also fail on π digits but fail in a quantitatively different way, which bounds the gap between gzip-K and true-K from below.

2. **K-accumulation vs S-cost for LLM training.** If a model is trained to fit text, the gzip ratio approximates how much of the training S-cost goes into learning K-structure vs being wasted on irreducible S-noise. Cross-entropy loss ≈ H(text) ≈ bits-per-token consumed; compression ratio ≈ K-content per token gained. A model achieving 2 bits/token loss on text with gzip ratio 0.30 is "spending" 2 S-bits to acquire ~0.6 K-bits. The remaining 1.4 bits are irreducible S-noise (random variation the model correctly learns is random). This can be computed on a small corpus without training a new model — just take an existing model's perplexity and compare to the corpus gzip ratio.

3. **Physical bound exploration.** The Bekenstein-Hawking entropy S = A/4 (in Planck units) bounds the number of distinguishable states. For concrete physical systems (hydrogen atom, neutron star, black hole), compute S and then ask: what gzip-K and true-K are consistent with those S values? This will show how large the K/S gap can be, making R1 concrete.
